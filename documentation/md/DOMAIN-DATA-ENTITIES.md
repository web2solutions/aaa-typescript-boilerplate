## Domain: Users

### Entity: `User`

Code references:

- `src/modules/Users/domain/Model/User.ts`
- `src/modules/Users/domain/Entity/IUser.ts`
- `spec/1.0.0.yml` (`components.schemas.User`)

| Field | Type (TS) | Format / Allowed values | Required | Validation expected |
|---|---|---|---|---|
| `id` | `string` | UUID string | Read model | Generated by `BaseModel` (`UUID.create/parse`). |
| `firstName` | `string` | free text, min length 2 in OAS | Create/Read | Domain: `canNotBeEmpty`. OAS: `minLength: 2`. |
| `lastName` | `string` | free text, nullable in OAS | Optional | Domain accepts empty string default. |
| `avatar` | `string` | file/path-like string | Optional | Domain: `canNotBeEmpty` when set. Default: `avatar.png`. |
| `username` | `string` | email format in OAS, min length 8 | Create/Read | Domain: `canNotBeEmpty`. OAS: `format: email`, `minLength: 8`. |
| `password` | `string` | password format in OAS | Create/Read (sanitized out in service responses) | Domain stores hash. Service enforces `mustBePassword` in creation/update password paths. |
| `salt` | `string` | free text (internal) | Internal | Internal cryptographic salt. Must never leak in external response. |
| `emails` | `EmailValueObject[]` | see `EmailValueObject` | Create/Read | OAS for create requires min 1 item. Domain creates via `createEmail`. |
| `documents` | `DocumentValueObject[]` | see `DocumentValueObject` | Optional | Managed via `create/update/deleteDocument`. |
| `phones` | `PhoneValueObject[]` | see `PhoneValueObject` | Optional | Managed via `create/update/deletePhone`. |
| `roles` | `string[]` | role slugs | Optional/Create/Read | Used by auth authorization checks. |
| `_createdAt` | `Date` (`serialized string`) | date-time | Read model | Managed by `BaseModel`. |
| `_updatedAt` | `Date` (`serialized string`) | date-time | Read model | Managed by `BaseModel`. |
| `_readOnly` | `boolean` (internal) | `true/false` | Internal | If true, mutating setters throw via `throwIfReadOnly`. |
| `_active` | `boolean` (internal) | `true/false` | Internal | Default true. |

## Domain Validation Rules (Cross-entity)

Shared validators in `src/shared/validators/index.ts` enforce rules used across entities/use cases:

- `canNotBeEmpty`
- `throwIfReadOnly`
- `mustBePassword`
- `throwIfPreUpdateValidationFails`
- `throwIfNotFound`

## Notes About Contract vs Domain Type Differences

Some fields intentionally pass through API boundary types and are normalized in domain constructors. Current known differences:

- `Phone.countryCode` and `Phone.localCode` are `integer` in OpenAPI but are stored as `string` in domain value object.
- `Document.type` includes `passport` in domain enum but OpenAPI document schemas list only `CPF`, `RG`, `SSN`.

When changing these contracts, update both:

1. domain entity/value-object type definitions
2. OpenAPI schemas (`spec/1.0.0.yml`)
3. tests covering creation/update paths
