# Users Domain Model (`Organization`)

## Purpose
`Organization` is the tenant aggregate in the Users domain. It models organization identity, communication channels, and membership references to users.

## Source of truth
- `src/modules/Users/domain/Model/Organization.ts`
- `src/modules/Users/domain/Entity/IOrganization.ts`
- `src/modules/Users/service/OrganizationService.ts`

## Construction
Constructor accepts `RequestCreateOrganization` + optional metadata.

| Field | Type | Default | Rule |
|---|---|---|---|
| `id` | `string` (UUID) | generated by `BaseModel` when missing | Parsed/normalized by `UUID.parse` when provided. |
| `createdAt` | `Date` | generated automatically | Auto-generated when entity is created. |
| `updatedAt` | `Date` | generated automatically | Auto-generated on create and auto-updated by persistence adapter on updates. |
| `name` | `string` | none | Required, `canNotBeEmpty`. |
| `address` | `AddressValueObject[]` | `[]` | Each entry mapped to `AddressValueObject`. |
| `phone` | `PhoneValueObject[]` | `[]` | Each entry mapped to `PhoneValueObject`. |
| `email` | `EmailValueObject[]` | `[]` | Each entry mapped to `EmailValueObject`. |
| `users` | `string[]` | `[]` | References to user ids in organization. |
| `readOnly` | `boolean` | `false` | Blocks mutable setters when enabled. |

## Relationships

- `@hasMany(() => User)` metadata is declared for `userEntities` to support adapter-level relation mapping while preserving domain contract with `users: string[]`.

## Aggregate invariants

- `name` cannot be empty.
- Value object arrays must contain valid value objects.
- Model payload and schema metadata are validated through OpenAPI 3.1 compliance helpers from `BaseModel`.
- Domain object arrays are mutated only through explicit methods:
  - `createAddress`, `updateAddress`, `deleteAddress`
  - `createPhone`, `updatePhone`, `deletePhone`
  - `createEmail`, `updateEmail`, `deleteEmail`

## Tenancy semantics

- `Organization` is the main tenant boundary for roles `admin` and `user`.
- User lifecycle operations synchronize organization membership references in service layer.
