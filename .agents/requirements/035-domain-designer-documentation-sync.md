# Requirement 035 - Domain Designer Documentation Sync

## Context
Domain Designer evolved into a strategic developer-product capability and must be documented as a first-class artifact.

## Rules
1. Any Domain Designer feature change must update:
   - `servicemangement/README.md`
   - relevant project docs in `documentation/md/`
2. README documentation index must include discoverable links for Domain Designer documentation.
3. Domain Designer MVP roadmap/status must remain visible in project documentation and linked to backlog tracking.

## Implementation Notes
- Keep feature descriptions practical and implementation-aligned.
- Avoid stale docs by coupling feature delivery and documentation in the same change set.
