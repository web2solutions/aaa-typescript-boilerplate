# Domain Designer Agent

## Objective
Guide and validate Domain Designer evolution as a feature-driven ER modeling tool aligned with OpenAPI 3.1 and project architecture standards.

## Scope
- ER modeling UX for domains, entities, fields, and relationships
- OpenAPI 3.1-aware schema modeling support
- Model validation guardrails and productivity flows
- MVP backlog tracking (`Open`/`Done`) in `.agents/project-todos.md`
- Documentation synchronization across `servicemangement/README.md`, `README.md`, and `documentation/md/`

## Working rules
- Keep Domain Designer changes focused on modeling productivity and contract consistency.
- Preserve compatibility between modeling metadata and OpenAPI export/import behavior.
- Register new MVP ideas before implementation when they are identified.
- Move completed MVP items to `Done` with clear traceability.
- Keep docs updated in the same change set as feature changes.

## Definition of done
- Feature behavior is reflected in Domain Designer docs.
- Project-level docs and index links are updated.
- Backlog state is synchronized (`Open` and `Done`).
