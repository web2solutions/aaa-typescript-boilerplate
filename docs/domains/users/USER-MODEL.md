# Users Domain Model (`User`)

## Purpose
`User` is the aggregate root of the Users domain. It encapsulates identity, profile data, credentials, roles, and child value objects (`emails`, `documents`, `phones`), and is responsible for enforcing mutation rules.

## Source of truth
- `src/modules/Users/domain/Model/User.ts`
- `src/modules/Users/domain/Entity/IUser.ts`
- `src/modules/port/BaseModel.ts`

## Construction
Constructor accepts `UserFactory` (internal type extending `RequestCreateUser`) and applies defaults:

| Field | Type | Default | Rule |
|---|---|---|---|
| `id` | `string` (UUID) | generated by `BaseModel` when missing | Parsed/normalized by `UUID.parse` when provided. |
| `firstName` | `string` | none | Required, `canNotBeEmpty`. |
| `lastName` | `string` | `''` | Optional. |
| `avatar` | `string` | `'avatar.png'` | Must not be empty when set. |
| `username` | `string` | none | Required, `canNotBeEmpty`. |
| `password` | `string` | `''` | Stored as hash by service layer. |
| `salt` | `string` | `''` | Internal cryptographic salt. |
| `emails` | `EmailValueObject[]` | `[]` | Added via `createEmail`. |
| `documents` | `DocumentValueObject[]` | `[]` | Added via `createDocument`. |
| `phones` | `PhoneValueObject[]` | `[]` | Added via `createPhone`. |
| `roles` | `string[]` | `[]` | Read-only after construction. |
| `readOnly` | `boolean` | `false` | Blocks all mutating setters/methods when `true`. |
| `active` | `boolean` | `true` | Internal activity flag. |

## Behavior methods

### Profile / credential mutation
- `firstName`, `lastName`, `avatar`, `username`, `password`, `salt` setters.
- All setters enforce `throwIfReadOnly` when aggregate is read-only.

### Nested object mutation
- `createPhone`, `updatePhone`, `deletePhone`
- `createDocument`, `updateDocument`, `deleteDocument`
- `createEmail`, `updateEmail`, `deleteEmail`

Each `update*` operation replaces the target item with a new value object instance (`new ...ValueObject({...old, ...payload})`) to keep nested rules centralized in value-object constructors.

## Read model access
Getters expose copied arrays:
- `roles`
- `emails`
- `documents`
- `phones`

This prevents direct external array mutation.

## Aggregate invariants
- Read-only mode prevents any mutation (`throwIfReadOnly`).
- Required user identity fields (`firstName`, `username`) cannot be empty.
- Nested entities must pass their own value-object validations at creation/update time.
- Domain model does not expose service concerns (HTTP, persistence, infrastructure adapters).
